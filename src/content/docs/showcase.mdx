## Custom Components

Here are the custom React components that can be used directly in MDX.

### Alert

Use alerts to convey important status messages to the user.

<Alert>
  <Terminal className="h-4 w-4" />
  <AlertTitle>Heads up!</AlertTitle>
  <AlertDescription>
    You can use the `<Alert>`, `<AlertTitle>`, and `<AlertDescription>` components together.
  </AlertDescription>
</Alert>

### Accordion

An accordion is perfect for hiding and showing content in a compact way.

<Accordion type="single" collapsible>
  <AccordionItem value="item-1">
    <AccordionTrigger>Is this an accordion?</AccordionTrigger>
    <AccordionContent>
      Yes. It's built using the `Accordion`, `AccordionItem`,
      `AccordionTrigger`, and `AccordionContent` components.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-2">
    <AccordionTrigger>Can I have another item?</AccordionTrigger>
    <AccordionContent>
      Absolutely! You can have as many as you need.
    </AccordionContent>
  </AccordionItem>
</Accordion>

### Image & AspectRatio

The custom `Image` component is from `next/image`. Here, it's wrapped in an `AspectRatio` component to maintain a 16:9 ratio.

<AspectRatio ratio={16 / 9} className="bg-muted mt-6 rounded-md border">
  <Image
    src="https://images.unsplash.com/photo-1588345921523-c2dcdb7f1dcd?w=800&dpr=2&q=80"
    alt="A beautiful landscape"
    fill
    className="rounded-md object-cover"
  />
</AspectRatio>

### Cards

Cards are great for grouping related content.

### Button

You can even use custom buttons directly in your content.

<Button>Click Me</Button>

### Steps

Create step-by-step guides easily with the `<Steps>` and `<Step>` components.

<Steps>
  <Step>Clone the repository</Step>
  <p>First, you need to clone the project from GitHub.</p>
  <Step>Install dependencies</Step>
  <p>Next, run `npm install` in the project directory.</p>
  <Step>Run the development server</Step>
  <p>Finally, start the server with `npm run dev` and you're good to go!</p>
</Steps>

### Tabs

Display content in a tabbed interface.

<Tabs defaultValue="account" className="w-full">
  <TabsList>
    <TabsTrigger value="account">Account</TabsTrigger>
    <TabsTrigger value="password">Password</TabsTrigger>
  </TabsList>
  <TabsContent value="account">
    This is the content for the **Account** tab. You can put any markdown or
    other components here.
  </TabsContent>
  <TabsContent value="password">
    This is the content for the **Password** tab.
    <Alert className="mt-4">
      <AlertDescription>
        Remember to use a strong, unique password!
      </AlertDescription>
    </Alert>
  </TabsContent>
</Tabs>
